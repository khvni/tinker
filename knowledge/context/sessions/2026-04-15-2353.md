## Scope

- Fix broken Google OAuth setup in feature 01 Better Auth flow
- Remove silent bad-client path
- Make provider redirect URI stable enough for Google/GitHub console config

## Changes

- Added strict Google/GitHub credential gating in Rust auth command layer
- Added env alias support for Better Auth standard names:
  - `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET`
  - `GITHUB_CLIENT_ID` / `GITHUB_CLIENT_SECRET`
- Added stable Better Auth callback port default `3147`
- Added explicit Better Auth provider `redirectURI` values:
  - `http://127.0.0.1:3147/api/auth/callback/google`
  - `http://127.0.0.1:3147/api/auth/callback/github`
- Stopped accepting placeholder or malformed Google client IDs in auth-sidecar config
- Improved Google runtime error mapping so `invalid_client` and `redirect_uri_mismatch` return setup guidance instead of opaque provider text
- Updated `.env.example` with real required vars and callback URI comments
- Regenerated bundled auth sidecar resource via desktop prepare script

## Decisions Made

- Stable Better Auth callback port beats random provider callback port. Reason: Google/GitHub console setup need deterministic redirect URI
- Keep random Rust loopback app callback for sidecar-to-app handoff. Reason: that path is local only and not vendor-registered
- Prefer canonical `*_OAUTH_*` env names in app, but accept Better Auth standard aliases for setup compatibility
