# Decisions Made

- Better Auth own browser OAuth flow. Rust keep only local plumbing. Reason: simpler than custom provider code, still respects keychain-only token storage
- Better Auth sidecar run locally as Node sidecar, not remote service. Reason: local-first and no app backend
- No database adapter for Better Auth. Use stateless cookie cache + account cookie for temporary browser session only
- Sidecar signs out before returning transfer ticket. Reason: Better Auth cookies must not become long-term token source
- Provider config now independent. Missing GitHub creds no longer blocks Google flow, and vice versa
