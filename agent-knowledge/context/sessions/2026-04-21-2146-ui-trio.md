# UI Trio — Modal + Toast + EmptyState primitives (TIN-177 / TIN-178 / TIN-181)

**Branch**: `khvni/ui-design-trio`
**PR**: draft (opened in this session)
**Scope**: Three `@tinker/design` primitives in one bundled PR — Modal, Toast, EmptyState — plus three renderer call-sites swapped from ad-hoc markup to `<EmptyState>`.

## Why bundled

Each primitive is ~100–300 LOC by itself. Shipping three 1-file PRs would mean three review cycles for work that shares tokens, folder conventions (D21), and the same smoke test. One PR lets the reviewer validate consistency across the trio in one pass.

## What landed

Three folder-per-component primitives under `packages/design/src/components/`, following D21:

```
Modal/
  Modal.tsx      Modal.css      Modal.test.tsx      index.ts
Toast/
  Toast.tsx      Toast.css      Toast.test.tsx      index.ts
EmptyState/
  EmptyState.tsx EmptyState.css EmptyState.test.tsx index.ts
```

Barrel exports added to `packages/design/src/components/index.ts`.

Three renderer sites upgraded to `<EmptyState>`:
- `apps/desktop/src/renderer/panes/Chat.tsx` — empty thread state (chat-bubble icon; description flips on `modelConnected`)
- `apps/desktop/src/renderer/panes/Today.tsx` — no indexed notes state (document icon + "Run memory sweep" action)
- `apps/desktop/src/renderer/components/IntegrationsStrip.tsx` — no providers connected state (link-chain icon, `size="s"`, `align="start"`)

Tokens extended in `packages/design/src/styles/tokens.css`:
- `--shadow-modal` (light + dark variants) — replaces ad-hoc shadow on the modal card
- `--color-scrim` raised to 0.55 (light) / 0.6 (dark) per Paper MCP dig

## Design decisions

- **Headless + no portal.** Modal renders as a DOM sibling in the provider tree; no `createPortal`. Reason: Tauri has a single webview, no iframe isolation, and z-index layering in `@tinker/panes` already reserves a scrim stratum. Portal is free complexity here. If we later need app-shell-level modals, one line promotes to `createPortal(…, document.body)`.
- **Toast provider is opt-in at the root, never nested.** Surface is `useToast()` hook + `<ToastProvider>`. No default global provider — consumers choose placement (bottom-right default). Keeps the primitive a pure library component; the app wires exactly one provider in `App.tsx` if/when it wants toasts.
- **EmptyState is a layout, not a widget.** No buttons, no state, no hooks. Just `{ icon, title, description, action }` slots. The three call-sites pass their own `<Button>` + SVG icon. Reason: empty states are domain-specific (memory vs chat vs integrations) — baking defaults would create three "almost right" versions.
- **`role` semantic split for toasts.** `error` → `role="alert"` (assertive, interrupts screenreader). Others → `role="status"` (polite). Matches WAI-ARIA toast pattern; no custom `aria-live` plumbing required.
- **Pause-on-hover uses mouseenter+focus / mouseleave+blur pair.** Keyboard users get the same pause behavior as pointer users. Pause clears the timer; resume starts a fresh one using remaining-time math so the toast doesn't reset its full duration on every hover.

## Paper MCP dig (pre-PR)

Paper file had a Modal spec (440px card, 14px radius, cream on warm-gray scrim, X close top-right). Three takeaways:

1. **Radius drift accepted.** Paper said 14px; we used `--radius-lg` (12px). 2px difference invisible at glance; keeping the token is worth more than the pixel.
2. **Shadow needed a token.** Paper's `0 20px 60px rgba(26, 22, 18, 0.25)` was too specific to hardcode into Modal.css. Added `--shadow-modal` in tokens.css with a darker dark-theme override so the shadow reads on warm-near-black without becoming a black hole.
3. **Scrim bumped.** Paper's scrim was visually denser than our `--color-scrim` of 0.38. Raised to 0.55 light / 0.6 dark. Affects every existing scrim user (model picker panel, future drawers) — reviewed: none relied on the previous low opacity as a design intent.

No Paper refs existed for Toast / EmptyState — designed from tokens + existing component palette (`StatusDot` semantic colors for toast variants; `Card` padding for EmptyState).

## What we didn't do (and why)

- **No `<ConfirmDialog>` wrapper.** Could build on Modal, but zero current callers need confirm semantics. Premature abstraction — D25 discipline.
- **No toast queue overflow UX.** `max` prop drops oldest silently. A "+3 more" pill was considered; current call-sites cap at 3, so the overflow path is unreachable in practice.
- **No EmptyState variants (error / loading / blocked).** One shape, one role. Adds friction only when a second shape actually lands.

## Follow-ups surfaced (not in this PR)

- EmptyState's dashed border looks too "placeholder" on the IntegrationsStrip card in light theme; may want a `variant="inline"` that drops the border when already inside a bordered container. Defer until a second inline call-site lands.
- Modal doesn't currently focus-trap into a `<select>` with an open native popup (Tab moves to next field instead of cycling inside the popup). Browser-native behavior; not in scope.
