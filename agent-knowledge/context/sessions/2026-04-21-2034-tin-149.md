# TIN-149 ‚Äî Scaffold `packages/workspace-sidebar`

## Scope

Ticket body: "Create `packages/workspace-sidebar`. Vertical list of workspace cards. Card: folder, branch, PR, ports, last-active. Drag-reorder."

Per feature spec [[13-workspace-sidebar]] (deferred post-MVP per [[decisions]] D25). Scaffold only ‚Äî not wired into `apps/desktop/src/renderer/App.tsx`. D25 permits deferred-feature code in-tree as long as it is not on the critical-path UI.

## What landed

- New workspace `packages/workspace-sidebar` (`@tinker/workspace-sidebar`):
  - `types.ts` ‚Äî `SidebarStatusEntry`, `WorkspaceCardModel`, `WorkspaceCardAttention`, `SidebarFlashAccent`, `SidebarStatusFormat`.
  - `WorkspaceSidebar/` ‚Äî top-level component. Drag-reorder via native HTML5 DnD + a custom MIME (`application/x-tinker-workspace-id`). Emits `onReorder(id, toIndex)`. `+` button for add; optional footer slot.
  - `WorkspaceCard/` ‚Äî one card per workspace. Renders `title` (folder), sorted `entries` rows (branch / PR / ports / last-active via `git.branch` / `git.pr` / `ports.*` / `last.active` keys), pinned marker, unread dot, and flash accent shadow from the attention model.
  - `styles/workspace-sidebar.css` ‚Äî consumer bundle (`@tinker/workspace-sidebar/styles.css`) that re-imports the two co-located component stylesheets.
  - `src/test-setup.ts`, `vitest.config.ts`, `tsconfig.json` mirror the `@tinker/design` test harness.

- Tests (`WorkspaceSidebar.test.tsx`, 7 cases):
  - one-card-per-workspace rendering + required metadata rows (branch / PR / ports / last-active).
  - pinned + unread a11y labels.
  - click activation.
  - add-button invocation.
  - footer slot rendering.
  - drag-reorder dispatches `onReorder(id, toIndex)`.
  - no-op when dropping onto the source.

- `pnpm-lock.yaml` updated with new package devDeps (`jsdom`, `@testing-library/react`, etc.).

## Design decisions

- **Tokens only** (D14/D23): all colors/spacing/radius/type read from `--color-*` / `--space-*` / `--radius-*` / `--font-*`; no hex, no `rgba()`.
- **Folder-per-component** (D21): `WorkspaceSidebar/` and `WorkspaceCard/` each contain tsx + css + index barrel; test co-located next to the shell component.
- **Native DnD, no extra deps**: HTML5 drag events are enough for a scaffold; avoids pulling in `react-dnd` / `dnd-kit` for a post-MVP surface that may reshape once wired to a real store.
- **Sort inside card**: `entries` sorted by `priority` ASC then `key` to match spec contract; caller can publish in any order.
- **Not wired into `App.tsx`**: intentional per D25. The package compiles + tests independently; integration is a later ticket when multi-workspace UX is in scope.
