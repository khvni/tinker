# Session — TIN-187 delivery

## What shipped

Single PR closing TIN-187. Boot path collapses to the workspace shell with a single Chat pane; the 3-step AI-model / Connected-tools / Vault panel is gone; the sign-in gate is deferred out of the boot path. Folder selection moves into the Chat composer via a new `<SelectFolderButton>` primitive next to `<ModelPicker>`.

## Scope delivered

1. **Removed**
   - `apps/desktop/src/renderer/panes/FirstRun.tsx` (fugly tri-panel layout).
   - `ONBOARDING_KEY` constant (and the `onboarded` state branch in `App.tsx`).
   - `signInGateVisible` / `workspaceAvailable` boot gates in `App.tsx` — workspace is always the root view.
   - `.tinker-first-run` + `.tinker-first-run-grid` CSS blocks in `styles.css` (defrag; no remaining callers).
   - `WEB_PREVIEW_NOTICE` export in `runtime.ts` (zero remaining callers after FirstRun delete).

2. **Added**
   - `@tinker/design/SelectFolderButton` primitive (D21 folder-per-component):
     - Paper `FolderPill` (E9-0) parity: pill shape, `--color-bg-elevated` ground, `--color-border-subtle` outline, 5/9 padding, 6 gap, 12×12 folder glyph, 10×10 chevron, `--font-size-sm` label.
     - Props: `folderPath?`, `loading?`, `placeholder?`, standard button attrs.
     - Loading state swaps chevron for `<Progress variant="spinner" size="xs">`.
     - Playground entry at `routes/design-system.tsx` (four states: empty / picked / loading / disabled).
     - 6 colocated tests (vitest + testing-library).
   - Chat composer header: `SelectFolderButton` slots right after `ModelPicker`. Empty state renders "Select a folder to start" CTA when the pane loads with no session folder.
   - `EmptyState.action` slot driving the primary "Select folder" button for the zero-state.
   - `App.tsx → handleSelectSessionFolder`: opens native picker → reads per-user memory subdir → restarts OpenCode inside the picked folder → re-initialises vault service + skill index → persists `tinker:vault-path` → flips `sessionFolderBusy` for the button spinner.

3. **Rust**
   - `restart_opencode` now accepts an `Option<RestartOpencodeOptions>` argument: `{ folderPath?, memorySubdir? }`. Backwards compatible — the existing renderer call site that omits args still resolves to `None`.
   - `bootstrap_opencode` reuses the same options shape and injects `--cwd <folder>` plus `SMART_VAULT_PATH` when supplied. The default path at setup stays at the opencode.json parent so the first cold launch still yields a working connection before the renderer replaces it.

4. **Session bootstrap behaviour**
   - No sign-in screen, no wizard. Cold boot → Workspace with single Chat pane.
   - If `localStorage.tinker:vault-path` exists, App restarts OpenCode into that folder before the first render completes, so last-active session reattaches automatically.
   - If no stored folder, Chat renders `<EmptyState>` pointing at the `SelectFolderButton` composer entry.
   - `DEFAULT_USER_ID = 'local-user'` continues to back `sessions.user_id` + `<memory_root>/local-user/` until M8 sign-in returns via Settings (deferred).
