## Architecture notes

- **Decoupled skill reinject from session reset.** `activeSkillsRevision` was pre-coupled to Chat's session-reset effect dep list — so any install / publish / toggle-active would abort the open OpenCode session, `setMessages([])`, dispose the history writer, and re-read from disk. Before this PR the revision only fired on vault reindex (rare). Installing from Playbook or publishing from Chat made the coupling user-visible. Fix: removed the revision from the reset effect's dep array and moved active-skill fetch to just-before-prompt via `injectActiveSkillsIfStale(sessionID:revision)`. Covered by new `Chat.skill-reinject.test.tsx`.
- **Shared auto-sync helper** (`skill-auto-sync.ts`) — one implementation, four early-return branches (git unavailable, no config, no root, happy), four dedicated tests. PlaybookPane + SaveAsSkillModal both route through it. No duplicated logic.
- **D25 compliance** — default boot layout untouched. Playbook reachable only via new header "Playbook" button in `Workspace.tsx` and the design-system playground.
- **D22 compliance** — skill store init passes config (`vaultPath`) per-call; re-init on user-id change. No stashed state.
