# Session — Skeleton + KeyboardHint primitives

## Scope

Two `@tinker/design` primitives shipped as one atomic PR.

- **TIN-186 / UI.11** — `<KeyboardHint>` kbd-style chip.

Both depend on UI.1 (which is done). Both are folder-per-component (D21), tokens-only (D14/D23), and exercised in the Components tab of the design-system playground.

## What was built

### Skeleton

- `variant: 'text' | 'circle' | 'rect'`, `width` / `height` props (number → px, string passthrough).
- Shimmer animation via token-based `linear-gradient` between `--color-bg-hover` and `--color-bg-elevated`; timing from `--ease-standard`.
- `prefers-reduced-motion: reduce` disables the gradient and animation.
- A11y: `role="status"`, `aria-busy="true"`, `aria-live="polite"`.
- Auto-mirrors `width → height` when `variant === 'circle'` and only `width` is provided.

### KeyboardHint

- `keys: string[]` → one `<kbd>` per key.
- Optional `os?: 'mac' | 'other'` prop; auto-detect via `navigator.platform` / `userAgent`, SSR-safe via `typeof navigator === 'undefined'` guard.
- Mac map: `Cmd→⌘`, `Alt→⌥`, `Shift→⇧`, `Ctrl→⌃`, `Enter→↵`, `Tab→⇥`, `Backspace→⌫`, `Delete→⌦`, arrows.
- Other map: spells modifiers out (`Cmd→Ctrl`, `Meta→Win`, …). Wide modifiers (e.g. `Shift`, `Enter`, `Ctrl`) get `tk-kbdhint__key--wide` for tighter padding.
- Single character keys uppercased. Unknown symbols pass through.
- `aria-label` composed from the displayed keys; explicit `label` prop wins.
