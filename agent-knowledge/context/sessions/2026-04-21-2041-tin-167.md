# TIN-167 — Release GitHub Action pipeline

Contributor: khvni
Agent: Codex

## Scope

- Deliver TIN-167 to PR-ready state: tag-driven GitHub Actions release pipeline for desktop bundles, macOS signing/notarization, GitHub Release upload, and updater manifest publishing.
- Address the out-of-band non-MVP infra ticket in `agent-knowledge/context/tasks.md` and hand off a PR for review.

## Changes

- Added `.github/workflows/release.yml` to create or reuse a draft GitHub Release on `v*` tags, then build macOS arm64, macOS x64, Windows x64, and Linux x64 bundles with `tauri-apps/tauri-action`.
- Added `apps/desktop/scripts/write-release-config.mjs` so CI can generate a temporary `src-tauri/tauri.release.conf.json` that enables updater artifacts and points builds at GitHub Releases `latest.json` without changing local desktop build behavior.
- Added `docs/release-pipeline.md` and linked it from `docs/README.md`.
- Updated `apps/desktop/src-tauri/.gitignore` to ignore the generated release-only config.
- Added an `Active non-MVP tickets` row in `agent-knowledge/context/tasks.md` and flipped TIN-167 to `review` after PR creation.

## Decisions Made

- [2026-04-21] Release-only updater config is generated in CI instead of committed as a permanent Tauri config file so local builds keep existing bundle behavior and the updater public key stays repository-configured.
- [2026-04-21] macOS arm64 and x64 builds run on separate native GitHub-hosted runners rather than cross-compiling on one runner; this avoids sidecar/node arch mismatches in the bundled Tauri binaries.
- [2026-04-21] The release matrix is serialized with `max-parallel: 1` so the final `latest.json` manifest is generated after all platform assets are present on the GitHub Release.
- [2026-04-21] Linux release builds target `ubuntu-22.04` to keep AppImage compatibility closer to the older baseline used by the official Tauri examples.
