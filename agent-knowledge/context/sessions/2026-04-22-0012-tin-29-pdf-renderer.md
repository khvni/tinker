# Session — TIN-29 PDF renderer integration

## Scope

Ship MVP PDF support for `FilePane` under M3.3: lazy-load the PDF renderer, render local PDFs inline, and support keyboard page navigation.

## What changed

- Added `pdfjs-dist@4.7.76` to `@tinker/desktop`.
- Registered `application/pdf` in `FilePane` and lazy-loaded a new `PdfRenderer` component so PDF code stays off the chat boot path.
- Built `PdfRenderer` under `apps/desktop/src/renderer/panes/FilePane/components/PdfRenderer/`:
  - local file bytes → `pdfjs-dist` document
  - canvas render scaled to pane width
  - page navigation via `Previous` / `Next` buttons and `PgUp` / `PgDn` / `Home` / `End`
  - worker configured through Vite URL import
- Updated `workspace/file-open.ts` so `.pdf` paths open as `application/pdf`.
- Expanded CSP in `tauri.conf.json` for the PDF worker (`worker-src 'self' blob:`) and PDF.js eval path (`'wasm-unsafe-eval'`).
