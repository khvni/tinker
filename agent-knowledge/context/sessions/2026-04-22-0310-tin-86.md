# TIN-86 — Per-user memory subdir creation on sign-in

## Outcome

PR #86 is open for review with the M8.13 slice.

## What landed

- Extended `packages/memory/src/memory-paths.ts` so the existing `memory.path-changed` pub/sub tracks both:
  - root moves (`previousRoot` / `nextRoot`)
  - active per-user subdir changes (`previousPath` / `nextPath`, `previousUserId` / `nextUserId`)
- Added `syncActiveMemoryPath(userId, { emit })` to:
  - create the active user's subdir on demand
  - emit only when the effective active user/path changes
  - stay no-op for repeat resolution of the same user
- Wired `apps/desktop/src/renderer/App.tsx` auth flows to:
  - validate refresh-token requirements before side effects
  - create the signed-in user's subdir on successful sign-in
  - re-sync the active memory path after auth status refreshes and OpenCode restarts
- Added focused tests covering:
  - no-op re-resolution for the same user
  - event emission when the active user changes
  - unchanged root-move event compatibility

## Verification

- `pnpm -r typecheck`
- `pnpm -r lint`
- `pnpm -r test`

## Notes

- This intentionally keeps the root-level Settings listener working unchanged; it still reads `nextRoot`.
- Downstream consumers that care about the active per-user directory can now read `nextPath` / `nextUserId` from the same event instead of inventing a second pub/sub.
