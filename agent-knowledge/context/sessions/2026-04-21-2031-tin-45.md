# TIN-45 — M4.8 Wire ModelPicker to OpenCode SDK providers

## Scope

- Deliver `agent-knowledge/context/tasks.md` row 4.8 from MVP pillar [[24-mvp-model-picker]].
- Bind real `ModelPicker` data to OpenCode SDK provider config without dragging in M4.9 persistence scope.

## Changes

- `apps/desktop/src/renderer/panes/Chat.tsx`
  - Added chat-surface `ModelPicker` next to the composer send button.
  - Loads provider/model rows from `client.config.providers()`.
  - Applies the selected `{ providerID, modelID }` to `session.prompt()` while leaving persistence for TIN-49.
  - Handles zero providers with the primitive's empty state instead of crashing or leaving stale selection state behind.
- `apps/desktop/src/renderer/opencode.ts`
  - Added pure helpers that flatten OpenCode provider config into picker rows, pick the initial selection from SDK defaults, and choose the first OAuth-capable provider without hardcoding `openai`.
- `apps/desktop/src/renderer/opencode.test.ts`
  - Added coverage for provider/model mapping, default-selection fallback, selected-row lookup, and OAuth-provider selection.
- `apps/desktop/src/renderer/App.tsx`
  - Deleted renderer-side `openai` assumption from model-connect checks.
  - OAuth connect flow now picks the first provider OpenCode exposes with an OAuth method and only enforces `https:` on the returned auth URL.

## Decisions Made

- 2026-04-21: Keep M4.8 in-memory only. Selection now affects outgoing prompts, but SQLite persistence and "new session inherits last used model" remain deferred to M4.9.
- 2026-04-21: Reuse the existing `@tinker/design` primitive exactly as shipped and do the SDK shaping in renderer helpers instead of expanding the primitive API.
- 2026-04-21: Remove the OpenAI-only assumption from renderer-side model discovery/auth because TIN-45 acceptance says provider/model data must come from OpenCode, not Tinker constants.
