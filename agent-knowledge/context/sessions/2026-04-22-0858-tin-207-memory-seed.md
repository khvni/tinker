# Session — TIN-207 Paper memory scaffold

## What shipped

- Added `packages/memory/src/memory-seed.ts` as the single source for the Paper `IQ-0` Memory demo scaffold.
- Deleted `apps/desktop/src/renderer/panes/MemoryPane/memory-preview.ts`; browser preview and design-system preview now read the shared seed module instead of ad-hoc Vite-only data.
- Empty memory-root bootstrap now seeds the shared demo notes only when the memory root is empty. Existing roots are left alone.
- `MemoryMarkdownFile` now carries `title`, `category`, `displayPath`, and `changesPreview` so the UI can match Paper: human title in sidebar/detail header, basename in file card, mock path in the card path row, and seeded `CHANGES` copy when git diff is unavailable.
- `MemoryPane` now auto-selects the Paper article note (`pending/synthesis-auto-20260408-glass-article.md`) and falls back to seeded `changes` content for non-git memory roots.
- Updated the Memory visual baseline to the new Paper-aligned seeded scaffold.

## Notes

- `apps/desktop/playwright.visual.config.ts` reuses an existing `dev:web` server when `CI` is unset. During verification, a stale server from another workspace on port `1420` served old preview data. Running the visual spec with `CI=1` forces a fresh server and avoids cross-workspace contamination.
