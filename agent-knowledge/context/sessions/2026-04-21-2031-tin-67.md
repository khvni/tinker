# TIN-67 — M7.2 Exa zero-config boot health check

## Outcome

Shipped a single Exa boot health check that runs once per OpenCode sidecar spawn, times out after 5 seconds, and stores the result in the renderer's integration-status state for follow-on M7.5 UI work.

## What shipped

- Added `apps/desktop/src/renderer/integrations.ts` with the Exa health-check helper, timeout guard, and status normalization.
- Added `apps/desktop/src/renderer/integrations.test.ts` covering connected, missing status, top-level SDK error, and timeout cases.
- Changed `apps/desktop/src/renderer/App.tsx` so provider-auth sync no longer performs MCP status polling. Exa status is now checked exactly once per sidecar connection change and written into `mcpStatus.exa`.
- Updated existing integration-status consumers (`IntegrationsStrip`, `FirstRun`, `Settings`, `Workspace`) to use the shared `MCPStatus` type and tolerate `checking` / `error`.

## Review loop

- Deleted dead GitHub MCP connect/disconnect calls from the boot sync path because `opencode.json` now contains only `qmd`, `smart-connections`, and `exa`.
- Kept the change scoped to Exa boot health. No broader M7.5 settings UI work landed here.
