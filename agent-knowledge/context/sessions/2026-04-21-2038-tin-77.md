## Changes
- Replaced `packages/auth-sidecar/src/main.ts`'s `/desktop/*` transfer shim with the ticketed `/auth/start`, `/auth/callback/:provider`, `/auth/session`, and `/auth/logout` flow.
- Stripped Google sign-in scopes down to identity-only (`openid`, `email`, `profile`) with offline refresh-token prompting; also removed GitHub's extra `repo` scope while touching the same file.
- Deleted the Rust one-off callback listener path in `apps/desktop/src-tauri/src/commands/auth.rs`; Tauri now starts auth with `POST /auth/start`, opens the returned local auth URL, and polls `/auth/session`.
- Wired refresh-token persistence through the M8.8 keychain bridge when a provider actually returns one, while keeping the existing session cache for current renderer code.
- Added Rust tests for parsing pending / ready / error `/auth/session` responses.

## Decisions Made
- [2026-04-21] Kept Tauri's public `auth_sign_in` / `auth_sign_out` commands intact and changed only their sidecar transport. This avoided widening the renderer surface during M8.4.
- [2026-04-21] Browser launch now targets the sidecar's own `GET /auth/start?ticket=...` URL instead of a direct provider URL so Better Auth can set its PKCE/state cookie before redirecting upstream.
- [2026-04-21] Refresh-token keychain writes are conditional. Google should populate one; GitHub may not, and M8.4 should not regress the existing GitHub sign-in path while 8.5 is still pending.
