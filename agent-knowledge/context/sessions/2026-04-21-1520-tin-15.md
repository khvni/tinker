## Design decisions

- **`userId: User['id']` (indexed access) over `userId: string`** — the ticket's second acceptance bullet ("Session.userId matches User.id shape") is enforced at the type layer, not by convention. When M8.2 replaces `User['id']` with a branded ID or broadens the field, `Session.userId` updates atomically — no drift possible without editing both sides.
- **`User` narrow placeholder** — ticket calls it a placeholder, M8.2 expands. Narrow means no speculative fields (`email`, `displayName`, etc.) to rip out when Better Auth's shape lands.
- **Timestamps as ISO strings** — matches existing `SSOSession.expiresAt` pattern. Keeps `@tinker/shared-types` type-only with zero runtime dependencies.
