# Session — TIN-84 Settings Account panel

## Outcome

Shipped M8.11: Settings pane "Account" section renders current user's avatar + display name + email + provider + sign-out button. Sign-out reuses existing `auth_sign_out` Tauri command, which already chains `clear_refresh_token(provider, user_id)` → `clear_session`. When the only active session disconnects, App's `signInGateVisible` flips true on the next render and returns the user to the SignIn route.

## What landed

- New component `apps/desktop/src/renderer/workspace/components/AccountPanel/` (folder-per-component per D21). `AccountPanel` is stateless — accepts `session`, `signOutBusy`, `signOutMessage`, `onSignOut`.
- New runtime `apps/desktop/src/renderer/workspace/settings-pane-runtime.ts` — React context + `useSettingsPaneRuntime()` hook mirroring the `ChatPaneRuntime` pattern. `pickActiveSession()` picks google → github → microsoft in priority order (documented in file; matches App's `pickCurrentUserId`).
- `SettingsPane.tsx` rewritten from empty stub to compose `SettingsShell` + `AccountPanel`. Account section is the first rail entry. Future Settings sections (Connections TIN-70/71/175, Memory TIN-60, etc.) slot in as additional `SettingsShellSection`s.
- `Workspace.tsx` wires `SettingsPaneRuntimeContext.Provider` around `PanesWorkspace`. Busy/message state for the active session is derived from per-provider props already threaded through `WorkspaceProps`.
- `register-pane-renderers.test.tsx` updated to provide the runtime context when rendering the Settings pane.
- `design-system.tsx` playground gets three new `Section`s demoing AccountPanel: signed-in, signing-out (busy + notice), and not-signed-in. The inline `Settings Shell > Account` SettingsShellSection also now renders `AccountPanel` with a neutral `demo@tinker.local` fixture.
- Legacy `apps/desktop/src/renderer/panes/Settings/` folder deleted (Settings.tsx + Settings.css + Settings.test.tsx + index.ts + useMemoryRootControls.ts). Grep confirmed zero external callers — it was zombie code from the pre-Dockview retirement. Removing it closes Reviewer's DELETE step concern before it accreted another parallel pane.

## Decisions

- **No new `useCurrentUser` hook in this PR.** M8.10 (TIN-83) owns the full hook contract with `users` table hydration. For TIN-84, a minimal `pickActiveSession(sessions)` helper reads the active SSO session directly from App state. TIN-83 replaces the derivation later without touching AccountPanel's API.
- **Runtime context pattern, not global store.** Followed `ChatPaneRuntimeContext` precedent. Keeps AccountPanel trivially testable in isolation (playground uses no runtime; production uses runtime). Aligns with D22 — no mutate-then-call managers.
- **Sign-out path uses `auth_sign_out`, not a new `auth_logout` endpoint.** The existing Tauri command already clears keychain + session. The ticket body says `POST /auth/logout` but the existing renderer→Rust surface is shorter and already in-repo; using it avoids scope creep.
