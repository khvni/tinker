# TIN-13 — M1.4 File pane MIME dispatch

Contributor: khvni
Agent: Codex

## Scope
- Deliver [[20-mvp-panes-workspace]] task 1.4 / Linear TIN-13 to PR-ready state.
- Keep scope to file-pane registration + dispatch only; do not churn `Workspace.tsx` or existing renderer internals before M1.7.

## Changes
- Added `apps/desktop/src/renderer/panes/FilePane/` with `FilePane.tsx`, `registerFilePane.tsx`, `index.ts`, and `FilePane.test.tsx`.
- Added a `mimeToRenderer` dispatch map that routes the legacy file renderers by MIME and falls back to an "unsupported, open externally" stub UI.
- Added `registerFilePane()` so the typed pane registry can register `{ kind: 'file', data: { path, mime } }` without touching the legacy Dockview workspace.
- Updated `agent-knowledge/context/tasks.md` M1.4 row to `review` with PR #38.

## Decisions Made
- 2026-04-21: kept M1.4 out of `Workspace.tsx`. The current workspace is still Dockview-backed; wiring the new registry into runtime now would mix M1.4 with M1.7 and violate the atomic-slice rule.
- 2026-04-21: adapted the existing Dockview-only renderers inside `FilePane` instead of editing those renderers directly. This contains the legacy prop-shape cast in one place and preserves the "no renderer changes in this issue" boundary from TIN-13.
- 2026-04-21: introduced temporary selector `text/markdown; mode=edit` for `MarkdownEditor` so the old editor surface stays representable in the MIME map until M3 replaces the edit flow.

## Knowledge Base Updates
- `agent-knowledge/context/tasks.md` updated to reflect PR review state for task 1.4.
- This session log added for future M1.7/M3 contributors so they know why `registerFilePane()` exists without a `Workspace.tsx` call site yet.

## Open Questions / Handoff
- M1.7 should decide where pane registrations are bootstrapped at runtime once `@tinker/panes` replaces Dockview in `Workspace.tsx`.
- M3 can expand `mimeToRenderer` with the new PDF/XLSX/DOCX/PPTX MIME types and remove the temporary markdown-editor MIME once editing has a non-Dockview path.

## Gaps Surfaced
- No shared pane-registration bootstrap module exists yet; each M1.x registration task currently needs to ship a helper and let M1.7 assemble the runtime wiring later.
