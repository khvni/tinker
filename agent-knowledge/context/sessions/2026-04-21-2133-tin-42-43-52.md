# 2026-04-21 21:33 — M4.5 / M4.6 / M4.15 bundled chat disclosures

## Scope

Bundled three sequentially-dependent M4 chat tickets in one branch (same files: `Chat.tsx`, `bridge/stream.ts`, design primitives). Splitting into three PRs would have meant two stub PRs immediately superseded by a third.

- **TIN-42 (M4.5)** — Collapsed tool-call disclosures, inline in message order, default hidden, expand to show name/args/result.
- **TIN-43 (M4.6)** — Reasoning blocks via the same disclosure pattern; ⌥T toggles all in the active chat.
- **TIN-52 (M4.15)** — Removed legacy `setStatus("Running …")` / `setStatus("X finished.")` lines and the parallel `toolResults` accumulator.

## Architectural changes

1. **Bridge `TinkerStreamEvent` extended.** Token / tool variants now carry `partID`. New `reasoning` and `tool_error` variants. `streamSessionEvents` tracks reasoning partIDs in a `Set` so text deltas (`message.part.delta` with `field === 'text'`) can be classified as text vs reasoning by looking up the prior `message.part.updated` for that part.

2. **Chat moved to folder-per-component (D21).** `apps/desktop/src/renderer/panes/Chat.tsx` → `apps/desktop/src/renderer/panes/Chat/{Chat,Block,draftReducer,draftReducer.test,index}`. Triggered by the file growing 1.7×; the moment to apply D21.

3. **Per-message `Block[]` model.** `ChatMessage.text: string` → `ChatMessage.blocks: Block[]` (text / reasoning / tool discriminated union, ordered, keyed by partID). `formatMessages` walks SDK parts → blocks. Streaming uses a pure `useReducer(draftReducer, [])` with an `upsertBlock(state, partID, build)` helper — every action variant is now 4–6 lines.

4. **`<Disclosure>` design primitive.** New `packages/design/src/components/Disclosure/`. Controlled or uncontrolled. Tone variants `tool` / `reasoning`. Tokens-only CSS.

5. **⌥T scope.** Keydown listener attached to the chat log container (via `tabIndex={-1}` on the scroller and `setLogScroller` callback ref) — NOT to `window`. Only fires when focus is inside the active chat pane's log; never steals OS keystrokes from textareas / inputs / contenteditable. Each Chat pane is independent.

## Reviewer pass

The first pass over-built: Chat.tsx +320 LOC, hardcoded `max-height: 320px` (D14 violation), global `window` keydown listener (would intercept `⌥T` everywhere including textareas → broke macOS `†` glyph entry). Reviewer (elon-reviewer) flagged 3 critical + 5 important.

Critical fixes applied in v2:

- **C1 keydown scoping** — moved listener to chat log container ref + `tabIndex={-1}`. No global listener; no OS keystroke theft; multiple panes independent.
- **C2 token compliance** — replaced `max-height: 320px` with `max-height: calc(var(--space-10) * 8)` + comment explaining the math. No new tokens added (D14 says feature PRs reuse existing tokens).
- **C3 reducer collapse + folder split** — `upsertBlock(state, partID, build)` helper makes every reducer arm 4–6 lines (was ~25 each). Reducer file: 63 LOC. Chat.tsx final: +110 LOC net (under the +120 target after the +320 v1 baseline).

Importants 4–8 also applied: collapsed `.tinker-tool-input/output/error` into one `.tinker-tool-pre[data-variant]`; inlined `placeholderTextForRole`; flattened `partToBlock` switch; renamed `allDisclosuresOpen` → `defaultDisclosureOpen` with doc comment on the resolution rule (`override ?? default`); single-pass `flatMap` for `toolResults` reconstruction.
