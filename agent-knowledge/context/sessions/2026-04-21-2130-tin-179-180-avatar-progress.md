# 2026-04-21 21:30 — TIN-179 + TIN-180 Avatar + Progress primitives

## Outcome

Two new tokens-only primitives in `@tinker/design`, bundled into one PR because they share the scaffolding (folder-per-component, playground rows, Paper references).

- `<Avatar>` (TIN-179): `src` + `name` + `size: xs|sm|md|lg`, deterministic palette fallback, image `onError` → initials.
- `<Progress>` (TIN-180): discriminated-union variant prop — determinate bar / indeterminate bar / spinner (xs/sm/md). Honors `prefers-reduced-motion`.

21/21 design-package tests pass; `pnpm -r typecheck` green across 10 workspaces.

## Why bundled

Both have the same shape (primitive in `packages/design`, playground rows in `design-system.tsx`, token-only CSS) and both are `S`/Urgent per Linear. Splitting into two PRs would double review overhead with no benefit — neither primitive blocks the other, both touch the same barrel export, both add non-overlapping rows to the playground. The bundled PR is well under the size where a split would help a reviewer.

## Paper-to-token mapping applied

Avatar:
- bg `#F4EFE4` → `--color-bg-hover` (theme-flipping).
- border `#1A16122E` → `--color-border-default` (hairline, both themes).
- pill radius → `--radius-pill`.
- initials Host Grotesk 600 → `--font-weight-semibold`, sized 10/11/13/16 to align md with `--font-size-base`.

Progress:
- determinate fill + spinner top stroke → `--color-accent` (amber, constant across themes per D23).
- track + spinner rest → `--color-border-subtle` / `--color-border-default`.
- spinner sizes 12 / 16 / 24 px (xs/sm/md) with borders 1.5 / 2 / 2.5 px.
- bar indeterminate: 40% fill, `translateX(-100% → 250%)` over 1.4s.
- spinner spin: 0.9s linear infinite.

## Decisions

- **Folder-per-component for both**, following ModelPicker precedent (D21). `packages/design/src/components/{Avatar,Progress}/{Component.tsx, Component.css, Component.test.tsx, index.ts}`. Grandfathered-flat primitives (Button, Badge, etc.) stay flat — new primitives go folder-per-component.
- **Deterministic palette via djb2-style hash** (`hash * 31 + charCode`, mod 6). Class-name based (`tk-avatar--p0..p5`) so it survives theme flips — the CSS var inside the class changes, the class doesn't.
- **Avatar sizes 20 / 24 / 32 / 40** — clean multiples of 4 aligned to the spacing scale. Fonts 10 / 11 / 13 / 16 keep readability at xs while lg picks up `--font-size-lg`.
- **Progress as a single component with discriminated union props** rather than two exports (`<ProgressBar>` + `<Spinner>`). The component is small, the variants share aria/role logic, and callers ask for "a progress indicator" regardless of shape.
- **`role="progressbar"` without `aria-valuenow` for indeterminate** per WAI-ARIA APG — no separate live-region needed.
- **No screenshot automation attempted**. The playground is a `pnpm --filter @tinker/desktop dev:design` Vite boot — brittle to capture from a headless tool in a fresh worktree. Reviewer runs the command and spot-checks; test plan lists the rows to exercise.
