# Session — TIN-188 guest-first boot

## Scope

- Remove the boot-time Better Auth gate so cold launch lands directly in Workspace.
- Add a persistent local `guest` identity for session FKs, memory scoping, and layout persistence.
- Surface consumer provider sign-in from Settings → Account, with `Continue as guest` as the primary action.

## What changed

- Deleted the Better Auth warm-start from `apps/desktop/src-tauri/src/lib.rs`; provider auth now starts only from explicit user action.
- Reworked `apps/desktop/src/renderer/App.tsx` to seed `guest`, restore any stored provider sessions, boot straight into `Workspace`, and support switching back to guest mode without relaunching.
- Updated `Workspace.tsx` to persist layouts by active `currentUserId` instead of the old hardcoded local user id.
- Replaced the placeholder Settings pane with a real Account surface under `workspace/components/SettingsPane/`, including a new nested `AccountSection/` folder (D21) and shared provider-picker guest CTA.
- Extended the SignIn route + ProviderPicker so the same provider controls can render `Continue as guest`.
- Recorded the product decision and PRD updates for guest-default boot (`D26`) plus the matching feature note changes.

## Verification

- `pnpm -r typecheck`
- `pnpm -r lint`
- `pnpm -r test`
- `cd apps/desktop/src-tauri && cargo test --lib`

## Notes

- Provider OAuth still depends on local client IDs/secrets being configured. Without them, Settings → Account shows the existing configuration error copy when a provider button is pressed.
