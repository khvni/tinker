# TIN-59 + TIN-61 + TIN-62 — Memory root seed, settings, and move flow

## Outcome

PR #69 is open for review with the M6.3 / M6.5 / M6.6 slice combined.

## What landed

- Extended `packages/memory/src/memory-paths.ts` with:
  - `getMemoryRoot()` to seed/read `app_settings.memory_root` without caching
  - `getActiveMemoryPath(userId)` to resolve `<root>/<user-id>/` and create it on demand
  - `validateMemoryRootWritable()` probe-file validation
  - `moveMemoryRoot()` with progress callbacks, empty-destination guard, rollback cleanup, and `memory.path-changed` pub/sub
- Wired `apps/desktop/src/renderer/App.tsx` to seed the memory root at boot and create per-user memory subdirs when auth resolves or a provider signs in.
- Refactored the active Settings pane into `apps/desktop/src/renderer/panes/Settings/` and added:
  - current memory-root display with truncation + tooltip
  - native folder picker action
  - toast-style success/error notice
  - modal progress overlay during moves
- Added test coverage for the new memory-path helpers and the Settings pane moving/notice states.

## Verification

- `pnpm -r typecheck`
- `pnpm -r test`

## Notes

- The move flow intentionally requires an empty destination folder. That keeps rollback simple and preserves the old root untouched when a copy fails.
- `memory.path-changed` is emitted from `@tinker/memory` as a lightweight pub/sub so M6.9 / M7.7 can subscribe without wiring the Settings pane directly to future consumers.
