## Non-obvious findings

- Better Auth's built-in `google` / `github` / `microsoft` providers enforce PKCE internally; there is no config toggle. `pkce: true` is generic-oauth-plugin-only.
- Microsoft's public-client flow works with Better Auth by leaving `clientSecret: ''` and requiring "Allow public client flows = Yes" in the Entra app registration. Confirms the feature-spec intent that M8.6 doesn't need a secret.
- Google + GitHub still issue a `client_secret` for Desktop / OAuth App client types. Google explicitly documents this is not treated as a true secret for installed apps. D4's intent ("no secret in binary") is preserved by loading via gitignored `.env.local`, not by refusing the value.
- The existing `packages/auth-sidecar/src/main.ts` scaffold has three divergences from the MVP contract: (a) grafts integration scopes onto sign-in (Gmail / Drive / repo) — strip to `openid email profile` + `read:user user:email`; (b) uses a ticket-shim endpoint surface (`/desktop/*`) — rename to `/auth/*`; (c) has no Microsoft provider. §9 of the doc captures the delta table M8.4–M8.6 must walk.
