## Decisions / deviations

- **Highlighter choice:** spec said "rehype-shiki" but parenthetical explicitly said "match existing code-highlighter choice." Existing = highlight.js via `renderers/code-highlighter.ts`. Chose to reuse directly via custom `pre` override rather than add a second highlighter or switch to shiki. Satisfies the "share code highlighter with M3.8/M3.9 — no second highlighter" acceptance criterion.
- **Streaming hook design:** extracted `createStreamingBuffer` as a pure factory so the buffer logic could be tested in node (no jsdom in desktop). Hook is a thin wrapper that owns the rAF/setState.
- **Name collision:** imported `ChatMessage` component collided with an existing `ChatMessage` type alias in `Chat.tsx`. Renamed local type to `ChatMessageRecord` — less churn than renaming the new component.
- **Exact-optional types:** `onSaveAsSkill` was being passed `undefined` for non-assistant rows, which trips `exactOptionalPropertyTypes`. Switched to conditional spread `{...(cond ? {onSaveAsSkill} : {})}`.
