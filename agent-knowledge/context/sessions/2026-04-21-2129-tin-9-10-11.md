# TIN-9 + TIN-10 + TIN-11 — legacy workspace shell retirement

## Outcome

Retired the legacy workspace shell in favor of `@tinker/panes`, removed the old panel dependency from the desktop app, and added a one-shot migration that drops incompatible pre-v1 layout snapshots. PR #62 is open and marked ready for review.

## What landed

- Rewrote `apps/desktop/src/renderer/workspace/Workspace.tsx` around `@tinker/panes` store + React workspace component.
- Replaced the old default layout with a single-chat `WorkspaceState<TinkerPaneData>` snapshot.
- Removed legacy workspace glue files and deleted the old deferred pane entrypoints that still depended on the retired shell.
- Refactored file renderers and `FilePane` to plain props so the workspace no longer needs legacy panel prop adapters.
- Renamed layout persistence to `workspace_state_json` and added a schema migration that drops incompatible legacy rows once, logging the drop count.
- Removed the deferred Playbook save flow and active-skill injection from chat so post-MVP skill-system behavior stays off the workspace critical path.

## Verification

- `pnpm install`
- `pnpm -r typecheck`
- `pnpm -r test`
- `grep -n "dockview-react" pnpm-lock.yaml` → no matches
- `find apps packages -type f \( -name '*.ts' -o -name '*.tsx' -o -name '*.css' \) | sort | xargs grep -nH "dockview\|Dockview"` → no matches

## Notes

- `main` did not contain the prerequisite M1.7 panes swap, so the PR carries M1.6 + M1.7 with TIN-9/10/11 instead of trying to stack the cleanup on top of missing work.
- `packages/bridge/src/skill-injector.ts` still exists for post-MVP work, but chat no longer calls it on the main workspace path.
- Manual `pnpm tauri dev` boot verification still needs a human run.
