## Changes
- Added `sessions` table creation to `packages/memory/src/database.ts` with `id`, `user_id`, `folder_path`, `created_at`, `last_active_at`, `model_id`, plus the `(user_id, last_active_at)` index.
- Included the `user_id -> users(id)` foreign key because `users` table from M8.3 is already present on `main`.
- Added `packages/memory/src/session-store.ts` exporting `createSession`, `listSessionsForUser`, `getSession`, `updateLastActive`, and `deleteSession`.
- Added focused tests in `packages/memory/src/session-store.test.ts` and extended `packages/memory/src/database.test.ts` to assert the new schema/index.
- Exported the new store from `packages/memory/src/index.ts`.

## Decisions Made
- [2026-04-21] Used direct CRUD helpers instead of a new store factory wrapper. Ticket only needed a small persistence slice, and `scheduler-store` was treated as a query-shape reference, not a reason to add new abstraction.
- [2026-04-21] Used ISO timestamp ordering directly in SQL (`ORDER BY last_active_at DESC`) instead of wrapping with `datetime(...)`, keeping the query aligned with the new `(user_id, last_active_at)` index.
