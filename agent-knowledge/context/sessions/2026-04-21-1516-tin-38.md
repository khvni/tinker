## Non-obvious findings worth remembering

- Selection is **pure client-side state** in OpenCode ‚Äî no SDK write on select, no `session.update()`. The chosen `{providerID, modelID}` is carried in the next prompt request. Tinker should preserve this property; don't introduce `config.set()` side-effects.
- Filter is `fuzzysort`, not substring ‚Äî the needle is lower-cased and scored across `["provider.name", "name", "id"]`. No match-highlighting.
- Loop keyboard nav is built in (`createList({ loop: true })`); Ctrl+N/Ctrl+P mirror arrow keys; Alt/Cmd+Arrow is explicitly bailed so native text-editing in the search input still works.
- "Latest" badge + default visibility use a per-provider/family newest-release heuristic clipped to the last 6 months (`packages/app/src/context/models.tsx:58-87`).
- `Escape` is deliberately short-circuited inside the list (`list.tsx:174`) so the popover / dialog owns the close handler.
- Popover is `modal={false}` with `placement="top-start"` and 4 px gutter ‚Äî surrounding UI stays interactive during pick.
- Provider icons are sprite-based with a `#synthetic` fallback glyph ‚Äî licensing check required before borrowing the sprite file directly.
