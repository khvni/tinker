# TIN-183 — Chat pane chrome redesign

## What shipped

- **Header bar** (`.tinker-chat-header`): ModelPicker moved up from composer to header left. Right cluster: `<ContextBadge>` (with `// TODO(TIN-174)` for real token plumbing), a `.tinker-chat-header__slot` for `modeToggleSlot` + `reasoningPickerSlot` props (both optional, `aria-hidden` when both missing), status `<Badge>`, and the ghost-style "New chat tab" button.
- **Framed message list** (`.tinker-chat-frame` wrapping `.tinker-chat-log`): `max-width: 720px`, centered, `padding: var(--space-5) var(--space-4)`. Webkit + firefox scrollbar styling using `--color-border-default` / `-strong` at `--radius-pill`.
- **Composer card** (`.tinker-composer-card`): border/radius-lg/shadow-sm container wrapping the existing `<Textarea>`. The inner Textarea is de-bordered and de-backgrounded via a descendant rule so the card owns the visual chrome. Focus-within border flips to `--color-accent-ring`; busy state uses `--color-border-strong`. Footer row carries an attachment-slot placeholder (`IconButton ghost` disabled + `aria-disabled` + paperclip SVG) on the left and Send/Stop on the right.
- **Playground**: `ChatTab` rebuilt to mirror the new chrome inside a 640 px `.ds-chat-frame`. Two sections — "Chat — with history" and "Chat — empty state (first message)". Dummy ModelPicker + ContextBadge + SegmentedControl (Chat/Plan) prove the slot wiring.

## Preserved (explicitly out-of-scope for TIN-183)

- `syncComposerHeight`, abort handler, `⌥T` disclosure toggle, memory injection, stream consumption, history window, skill injection, draft reducer, `ChatMessage` markdown rendering, `MessageBlock` disclosures — all untouched.
- `EmptyState` stays as-is; it just lives inside the new frame.

## Decisions

- **Tokens-only CSS.** Every value maps to a `--*` from `packages/design/src/styles/tokens.css`. Zero hex literals / bare px in the diff (verified via grep). Composer card paddings use `var(--space-3)` / `var(--space-2)` / `var(--space-4)`; radius-lg; shadow-sm.
- **ContextBadge is static this ticket.** Percent hard-coded to `57` with `tokens = windowSize * 0.57`. TODO comment ties to TIN-174. Picks up real values when reasoning/context streaming data flows in.
- **ContextBadge only renders when `selectedModel` exists** — keeps the header clean on a fresh `No model connected` state. Tested.
- **`.tinker-pane--chat` modifier** disables the default `padding` / `gap` on `.tinker-pane` so the chat chrome can own the full-bleed header border + framed log + composer bottom padding.
- **Slot container semantics.** Always rendered (single DOM node) but carries `aria-hidden="true"` when both props are absent, so AT skips an empty container.
- **`.ds-chat-frame` playground wrapper** gives the preview a bounded 640 px height so the log scroll + composer layout are visible side-by-side in one section. Real Chat pane uses the workspace pane shell's 100% height.
