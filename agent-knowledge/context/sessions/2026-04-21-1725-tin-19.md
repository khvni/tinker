# TIN-19 — `stop_opencode` Tauri command

## What shipped

Implemented M2.5: `stop_opencode(pid: u32)` Tauri command for folder-scoped OpenCode sidecar shutdown. SIGTERM → 2s graceful wait → SIGKILL fallback → manifest sweep by pid. Idempotent on dead pids and missing manifests. Rejects `pid == 0` at the command boundary (`kill(2)` broadcast footgun).

## Files changed

- `apps/desktop/src-tauri/src/commands/opencode.rs` — `stop_opencode` + `stop_opencode_inner` + unix signal helpers (`process_alive`, `send_term`, `send_kill`) + `remove_manifest_for_pid`. 7 new tests (13 total cargo lib tests).
- `apps/desktop/src-tauri/src/lib.rs` — registered command; renamed legacy free fn `stop_opencode(&AppHandle)` → `terminate_legacy_opencode` (reviewer flagged shared-name hazard).
- `apps/desktop/src-tauri/Cargo.toml` — `libc` (unix-target only), `tokio` `macros` + `rt` features.
- `apps/desktop/src/bindings.ts` — `stopOpencode(pid)` TS wrapper.

## Reviewer cuts (elon-reviewer, code-reviewer)

- Dropped the Windows signal branch. Windows isn't in MVP scope (D25, mac-first). Added `compile_error!` for non-unix targets.
- Deleted the post-SIGKILL 200ms settle sleep — kernel reaps synchronously, no observable effect.
- Renamed the legacy helper to kill the latent shared-name hazard.
- Kept `stop_opencode_inner` split vs. inlining: justified because `AppHandle` is impractical to construct in unit tests, and the extra surface is three lines.

## Tests

`cargo test --lib` — 13/13 green. New tests:
- `remove_manifest_for_pid_removes_matching_only` / `_is_idempotent_when_no_match` / `_ignores_missing_dir` / `_skips_malformed_files`.
- `stop_opencode_inner_is_idempotent_for_dead_pid` — spawns + reaps `true`, then stops the now-dead pid.
- `stop_opencode_inner_terminates_live_process_and_removes_manifest` — spawns `sleep 30`, SIGTERM path, asserts process gone + manifest removed.
- `stop_opencode_inner_force_kills_process_that_ignores_sigterm` — shell that `trap '' TERM; sleep 30`, asserts SIGKILL fallback kills within the 2s window.

`pnpm -r typecheck` + `pnpm -r test` — green across all workspaces.

## Gotcha

First draft used `pid == 0` as the "never-alive" pid in the idempotency test. `libc::kill(0, sig)` **broadcasts to the calling process group** — that killed the cargo test runner mid-run and returned a spurious exit 144. Fix: spawn `true`, reap it, and use that pid. Boundary guard on `pid == 0` added to the command itself so a renderer bug can never hit this path.
