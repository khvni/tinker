# Session — TIN-201 visual parity gate

## What shipped

- Playwright + pixelmatch harness in `apps/desktop`:
  - `playwright.visual.config.ts` — single worker, serial, Chromium, 1440×900, `deviceScaleFactor: 2`, `dev:web` webServer.
  - `playwright/visual/compare.ts` — `pixelmatch` + `pngjs` helper. Missing baseline → seed + warn. `UPDATE_BASELINES=1` → overwrite.
  - `playwright/visual/workspace.spec.ts` — screenshots guest boot workspace shell.
  - `playwright/visual/memory.spec.ts` — clicks sidebar Memory, screenshots pane.
  - `playwright/visual/settings.spec.ts` — clicks sidebar Settings, screenshots Account + Connections.
- Baselines committed: `boot-guest.png`, `memory-pane.png`, `settings-pane.png` under `apps/desktop/test-assets/baselines/`.
- CI: `visual` job added to `.github/workflows/ci.yml` after `ci` job. Artifact upload on failure only, name includes `run_id`, retention 7 days.
- Docs: `agent-knowledge/reference/visual-testing.md` explains code-baseline vs Paper-diff, local run instructions, baseline update flow, Paper human-reference usage, and follow-up ticket note.
- `tasks.md`: TIN-201 row added under Testing infra, status `review`, PR #115.

## Scope cuts / deviations

- **Paper PNG as CI oracle**: rejected. Font rasterization, scrollbars, and export dialog make Paper-PNG diff infeasible. Code-baseline approach adopted.
- **Label-based opt-out bypass (`visual-change-approved`)**: out of scope. Documented as follow-up TIN-201-b to be filed in Linear.
- **Agents pane**: explicitly out of scope per ticket.
- `pnpm dev:web` used instead of `pnpm dev:desktop` because Tauri needs native GTK/webkit on Linux CI.
- `.github/workflows/ci.yml` used instead of ticket's `typecheck-and-test.yml` (file does not exist).
