# TIN-12 — M1.3 Register chat pane kind

## Summary

Shipped the M1.3 chat-pane registration slice from [[20-mvp-panes-workspace]] with a boot-time `registerPane('chat', ...)` call and no `Chat.tsx` internals changes.

## What landed

- `apps/desktop/src/renderer/main.tsx` now calls `registerWorkspacePanes()` at boot.
- `apps/desktop/src/renderer/workspace/register-panes.tsx` registers the `chat` pane and is idempotent across repeated boot calls.
- `apps/desktop/src/renderer/workspace/components/RegisteredChatPane/` adds a thin wrapper that reuses the existing `<Chat>` component unchanged.
- `apps/desktop/src/renderer/workspace/chat-pane-runtime.ts` defines the chat runtime context used to supply current `Workspace` chat props to the registered renderer.
- `apps/desktop/src/renderer/workspace/Workspace.tsx` now provides that runtime context around the current Dockview tree so M1.7 can swap shells without reworking chat internals.
- `apps/desktop/src/renderer/workspace/register-panes.test.tsx` covers bootstrap registration and idempotent re-entry.

## Review notes

- Stayed inside TIN-12 scope: pure registration plus the smallest bridge needed to keep `Chat` unchanged.
- No `Chat.tsx` edits, no new pane kinds, no `@tinker/panes` shell swap, no Dockview cleanup.
- Context bridge chosen over prop-mutation singleton to stay compatible with D22-style per-render config flow.
