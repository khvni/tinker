# 2026-04-22 14:45 — Chat textbox port (anomalyco parity)

## Goal
User flagged current chat composer as jank. Asked to port anomalyco/opencode
`packages/app/src/components/prompt-input` visual pattern — single card with
attach icon + send arrow, then chips below for mode, model, thinking level.

## What shipped
- New primitives in `@tinker/design`:
  - `PromptComposer` — card wrapper (textarea + attach-left + send-right), auto-size,
    busy-swaps send for stop, optional `notice` + `controls` slots below card.
  - `ComposerChip` — pill trigger styled like `ModelPickerTrigger` for reuse.
  - `Menu` — minimal popover list with selection + Escape/outside-click close.
- `apps/desktop/src/renderer/panes/Chat/Chat.tsx`
  - Deleted `modeToggleSlot` / `reasoningPickerSlot` props (never wired upstream).
  - Header now just holds status badge, context badge, folder picker, new-chat button.
  - Below composer: Mode (Build/Plan) chip → ModelPicker → Thinking chip
    (Default/low/medium/high/xhigh).
  - Wired `agent` + `variant` into `client.session.prompt` so OpenCode actually
    receives the choice.
- `shared-types`: extended `REASONING_LEVELS` to 5-step scale incl.
  `default` + `xhigh`; `DEFAULT_REASONING_LEVEL` = `'default'`.
- `opencode.ts` — expanded `REASONING_VARIANT_ALIASES` for the new levels.
- `design-system` showcase updated to use `PromptComposer` + chips (drops the
  old `tinker-composer-card__body/footer` markup) + dropped unused
  `tinker-chat-header__slot` CSS.

## Skipped
- Did **not** port anomalyco's full `PromptInput` (contenteditable, slash-popover,
  file paste, drag overlay, agent mentions). Out of scope for this jank fix —
  our existing `Textarea` + chat state was already functional.
