# Session — TIN-73 minimal MCP connection gate

## What shipped

- Added a chat-local MCP readiness gate for brand-new sessions only.
- Reused the existing `@tinker/design` `<ConnectionGate>` primitive instead of introducing new shell-level boot UI.
- Polled OpenCode MCP status for `qmd`, `smart-connections`, and `exa` until all connected, an error occurred, or 10 seconds elapsed.
- Kept the composer disabled while the gate was active.
- Added error-state actions: `Retry` and `Skip for now`.
- Added timeout / skip notices so the composer can open with a warning instead of deadlocking the session.
