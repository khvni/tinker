# TIN-8 + TIN-14 — default layout + workspace engine swap

## Outcome

Moved the active workspace render path off Dockview and onto `@tinker/panes`. PR #61 is open and marked ready for review.

## What landed

- Replaced `layout.default.ts` with `createDefaultWorkspaceState()`, returning a fresh `WorkspaceState<TinkerPaneData>` seeded with one Chat pane.
- Rewrote `apps/desktop/src/renderer/workspace/Workspace.tsx` to mount `@tinker/panes`'s `<Workspace>`, hydrate/persist `WorkspaceState` snapshots through `@tinker/memory/layout-store`, and keep only chat/file/settings/memory in the active workspace path.
- Ported `chat-panels.ts` and `file-open.ts` from Dockview panel APIs to `@tinker/panes` store actions.
- Updated `packages/shared-types/src/layout.ts` and `packages/memory/src/layout-store.ts` to persist version-2 `WorkspaceState<TinkerPaneData>` snapshots instead of Dockview JSON.
- Boot registration now includes settings + memory placeholders, and the old Dockview-only CSS hook in `styles.css` was deleted.
- Removed deferred/broken Dockview-only hooks from the active MVP path: Chat no longer opens Playbook from the message list, and Markdown renderer no longer exposes the dead "Edit note" button.
