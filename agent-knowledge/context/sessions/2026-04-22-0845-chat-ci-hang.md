# 2026-04-22 08:45 — Chat CI hang

## Goal
Fix the failing GitHub `Typecheck and test` action after the desktop Vitest job
started getting cancelled instead of reporting an assertion failure.

## What shipped
- `apps/desktop/src/renderer/panes/Chat/Chat.tsx`
  - Stabilized the MCP status loader passed into `useMcpConnectionGate` with
    `useCallback`.
  - This stops the hook from restarting on every render once the gate is
    enabled, which was leaving the desktop Vitest process alive in CI.
- `apps/desktop/src/renderer/panes/Chat/Chat.skill-reinject.test.tsx`
  - Updated the mock client to include `mcp.status()`.
  - Added the current `updateSession()` memory export so the test matches the
    live `Chat` dependency surface.

## Root cause
- The attached CI log showed `pnpm -r typecheck` passed and desktop tests went
  green, but the job was cancelled during cleanup because Vitest never exited.
- Local repro matched CI: `CI=1 pnpm --dir apps/desktop exec vitest run` passed
  all 61 test files and then hung.
- Bisecting the renderer suite isolated the leak to
  `Chat.skill-reinject.test.tsx`.
- That test enables the MCP connection gate path; because `Chat` passed a fresh
  `loadStatus={() => client.mcp.status()}` function on every render, the hook
  dependency changed continuously and the check loop kept restarting.

- `pnpm -r typecheck`
