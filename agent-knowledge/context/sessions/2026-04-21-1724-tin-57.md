# TIN-57 — M6.1 app settings store

## Shipped

Added `app_settings` to the SQLite bootstrap schema in `packages/memory/src/database.ts`, then introduced `packages/memory/src/settings-store.ts` as the shared JSON config surface for upcoming memory-root work. The store exposes `list`, `get`, `set`, and `delete`, hydrates rows into typed values, and drops malformed JSON rows with a warning instead of crashing. `packages/memory/src/index.ts` now re-exports the store.

Added `packages/memory/src/settings-store.test.ts` to cover row hydration, invalid JSON handling, ordered listing, keyed fetch, upsert SQL, and delete SQL.

## Reasoning

TIN-57 maps cleanly to M6.1 in `agent-knowledge/context/tasks.md` by the existing Linear-to-task numbering pattern already used across TIN-5/TIN-6/TIN-15/TIN-17/TIN-18/TIN-27/TIN-38/TIN-53. Work stayed inside the task boundary: no memory-root resolution, no settings UI, no path-move logic, no event bus. That keeps M6.2+ unblocked without pulling later memory behavior into this PR.
