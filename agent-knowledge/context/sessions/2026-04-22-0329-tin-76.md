# TIN-76 — M8.3 SQLite users table + user-store

## Outcome

Shipped the `users` persistence slice for MVP identity. PR #36 is open and marked ready for review.

## What landed

- Added `users` table creation to `packages/memory/src/database.ts` with columns mirroring `User` plus a unique composite index on `(provider, provider_user_id)`.
- Added `packages/memory/src/user-store.ts` exporting `upsertUser`, `getUser`, `getUserByProvider`, `updateLastSeen`, and `hydrateUserRow`.
- Exported the new store from `@tinker/memory`.
- Wired the existing native sign-in path in `apps/desktop/src/renderer/App.tsx` to call `upsertUser` after a successful `auth_sign_in`.
- Added tests covering schema presence, signed-in user insert, repeat sign-in upsert semantics, and `last_seen_at` updates.

## Verification

- `pnpm -r typecheck`
- `pnpm -r test`

## Notes

- `SSOSession.userId` from the Better Auth sidecar is the provider account id (`account.accountId`), so the renderer stores local user ids as `${provider}:${providerUserId}` and keeps `provider_user_id` for lookup/upsert.
- Re-sign-in preserves the original `created_at`; only profile fields and `last_seen_at` update on conflict.
